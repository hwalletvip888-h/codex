# Risk Control

## 钱包安全

- 永不提交 `.env`、私钥、助记词、API Secret。
- 生产钱包和测试钱包分离。
- 新策略先用小额或测试网验证。
- 授权额度尽量最小化，并定期检查和撤销。
- Solana 钱包和 EVM 钱包分开管理，SOL meme 策略优先使用专用小额钱包。

## 策略风险

- 记录每个机会的资金占用、预期周期、退出条件和最大可接受亏损。
- 不把单一协议、单一链或单一资产作为全部资金暴露。
- 对空投、积分、质押、LP、借贷和桥接分别记录不同风险。
- Solana meme 币单笔仓位必须小，默认视为高波动、高归零概率资产。
- 合约未验证、持币过度集中、流动性过低、税费异常、疑似貔貅盘时直接跳过。
- 新币早期交易优先关注能否卖出，而不是只看涨幅。
- Mint authority、freeze authority、dev wallet、Top holders、bundle / sniper 痕迹异常时直接拦截。

## 执行风险

- 涉及真实资金的操作必须二次确认。
- 自动化流程默认只做提醒和草稿，不直接广播交易。
- 广播交易前检查链 ID、合约地址、方法名、金额、滑点和 Gas。
- 买入前必须确认 token 地址，不通过名称或 ticker 下单。
- 卖出路径必须提前验证，不能等到价格异动后再找退出方案。
- Solana 交易必须确认 mint address、pool address、路由、滑点和预期收到数量。

## 复盘字段

- 机会来源
- 执行时间
- 投入资产
- 交易哈希
- 手续费
- 收益或积分变化
- 风险事件
- 下一步动作
- 入场理由
- 退出理由
- 是否按计划执行
